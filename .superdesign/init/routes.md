# Route Map

The route map is specified but not implemented.

| URL | Surface | Layout | Purpose |
|---|---|---|---|
| `/planlegg` | Planlegg basket | Handleplan shell | Add needs and exact products, quantities, constraints, and optional location. |
| `/planlegg/resultat` | Plan results | Handleplan shell | Compare complete Pareto-frontier plans through the convenience/savings selector. |
| `/planlegg/handle` | Mobile checklist | Focused Handleplan shell | Shop the selected plan in route order with store-grouped items and offline tolerance. |
| `/oppdag` | Oppdag | Handleplan shell | Browse explainable basket-relevant offers, price drops, and plan impact. |
| `/review` | Private review | Authenticated operations shell | Review uncertain extracted retailer offers with source crop and audit actions. |

The first public visual flow should cover the first four routes. The private review surface is secondary but should reuse the trust and status primitives.
