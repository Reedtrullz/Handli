# Shared UI Components

## Repository state

Handleplan is a design-only greenfield repository. No frontend framework or component library has been selected, and no implementation components exist yet. The first Superdesign draft must establish reusable visual patterns without implying that source components already exist.

## Planned primitives from the approved specification

- Button: primary, secondary, quiet, and destructive review actions.
- Text input / product combobox with quantity and match-confidence affordances.
- Card for basket items, plans, offers, store groups, and trust explanations.
- Segmented control for Planlegg/Oppdag and transport mode.
- Accessible discrete plan selector presented as both a slider and named plan cards.
- Status badge for freshness, geographic scope, membership, multibuy, and substitution state.
- Checkbox row for the mobile shopping checklist.
- Disclosure for provenance, assumptions, and price conditions.

There is no source code to include because implementation has not started.
