## Deployment state

- The protected preview is not a public v1 launch and must remain protected while any release gate is not passed.
- The deployed application predates the uncommitted v1 foundation migrations and worker; this record makes no deployment claim for those changes.
- `PRICE_EVIDENCE_READ_MODEL` defaults to the legacy read model. Evidence/shadow modes require explicit configuration and eligibility proof.
- Production migration requires a pre-deploy database backup and a verified rollback/readback sequence.

## Explicit non-claims

This baseline does not prove:

- rights to operate any source publicly;
- complete Bunnpris, REMA 1000, or Extra coverage;
- official discounts, official before-prices, branch inventory, or branch-specific shelf prices;
- g/ml/piece fulfilment in the user-facing planner;
- travel routing, origin privacy, or travel-aware frontier correctness;
- an offline Handlemodus/PWA journey;
- public security, legal, accessibility, backup/restore, monitoring, or governance readiness; or
- a public v1 release.

The current decision is therefore **continue protected implementation; public launch blocked**. The live decision source is the [release-gate ledger](../../release/v1-release-gates.md).
